# Expense Counter — промпт для Claude Code

## Описание приложения
Минималистичное приложение для подсчёта расходов для мобильных устройств.

## Стек технологий
- React Native
- SQLite

## Визуальный референс
В приложении есть интерактивный прототип дизайна (Claude Design, файл `.dc.html`).
**Использовать его только как визуальный референс** — цвета, отступы, округления, размеры шрифтов, форма и расположение кнопок, компоновка экранов. Всю функциональную логику (взаимодействия, работа с БД, валидация, edge-кейсы) реализовывать строго по разделу "Функции и логика взаимодействия" и "Структура и логика приложения" ниже — прототип её не отражает и не является источником истины по поведению.

## Дизайн, UI/UX

### Стиль
Минимализм. Плоский дизайн: без теней, без градиентов. Крупный радиус скругления (16–20px) у кнопок и карточек. Много воздуха, свободного пространства.

### Шрифт
Nunito.

### Цветовая палитра
- `#F7F1DE` — фон приложения (60%)
- `#4E220F` — основной текст (30%)
- `#B0BA99` — акцентный цвет, используется для второстепенных элементов и неактивных состояний (10%)
- `#6B8F5C` — цвет положительной суммы / режима "добавка" (доход)
- `#B5533C` — цвет отрицательной суммы / режима "вычет" (расход)
- `#FBF7EA` — цвет кнопок в сетке цифр (немного светлее фона), плоские, без обводки

### Верхняя панель (шапка)
Минимальная высота.
- **Центр:** пилюля (pill-переключатель), разделённая на 2 сегмента: "Счётчик" и "История". Активный сегмент визуально выделен (заливка акцентным цветом, текст основного цвета на фоне акцента).
- **Правый край:** иконка настроек (шестерёнка).

### Экран "Счётчик"

#### Блок счётчика (под шапкой)
Две строки чисел, выравнивание по центру:
1. **Остаток** — крупный размер шрифта, цвет `#4E220F`.
   - Отображается с копейками/центами по умолчанию (см. настройки округления ниже).
   - Может уходить в отрицательные значения — это допустимо, специальных ограничений/предупреждений не требуется.
2. **Значение текущей вводимой части** — меньший размер шрифта.
   - Режим "вычет" — цвет `#B5533C` (красный)
   - Режим "добавка" — цвет `#6B8F5C` (зелёный)

#### Текстовое поле
Текстовое поле в одну строку над блоком кнопок — для названия операции (необязательное).

#### Блок кнопок — сетка 4×4
Кнопки скруглённой формы, цвет `#FBF7EA`.

| 7 | 8 | 9 | Смена режима |
| --- | --- | --- | --- |
| 4 | 5 | 6 | Удаление |
| 1 | 2 | 3 | Введение ↵ |
| 0 ↔ | 0 ↔ | . | Введение ↵ |

**Особые кнопки:**
- **Смена режима** (1 ячейка, ряд 1): переключение между "вычет" и "добавка". Иконка — два равнобедренных треугольника (красный основанием вверх, зелёный основанием вниз), сменяющие друг друга. Анимация переключения: вращение с плавным перетеканием цвета между красным и зелёным.
- **Удаление** (1 ячейка, ряд 2): иконка Backspace.
- **Введение** (растянута по вертикали на 2 ячейки, ряды 3–4): иконка Enter.
- **0** (растянута по горизонтали на 2 ячейки, ряд 4).
- **.** (обычная кнопка, ряд 4).

### Экран "История"
Отдельный экран (второй сегмент пилюли). Список прошлых транзакций:
- Дата операции
- Сумма (цвет по типу: зелёный для добавки, красный для вычета)
- Группировка по датам ("Сегодня", "Вчера", далее — дата в формате ДД.ММ.ГГГГ)
- Элемент списка:
  - С правой стороны — сумма.
  - С левой стороны:
    - Если у сделки есть имя: имя сделки сверху (крупный жирный шрифт), дата и время — снизу (мелкий тонкий шрифт).
    - Если имени нет: дата и время сверху, под ними — кнопка быстрого добавления имени (квадрат со знаком "+", тонкая рамка со скруглёнными углами, акцентный цвет).
  - Знак и цвет суммы:
    - Вычет — перед числом знак "−", цвет `#B5533C`.
    - Добавка — цвет `#6B8F5C`, без знака или со знаком "+" (на усмотрение реализации, главное — цвет).
    - Изменение баланса (`type = total`) — имя автоматически "Изменение баланса", цвет обычного текста `#4E220F`.
- Элементы списка разделены горизонтальной линией.
- Верхняя часть экрана — кнопка "Фильтр и сортировка" справа, иконка воронки.
  - **Модальное окно фильтра** — список из двух секций:
    - Тип записи: "Все", "Доход", "Расход", "Изменение баланса".
    - Дата записи: "Начало периода", "Конец периода".

#### Модалка изменения/удаления сделки
Открывается долгим нажатием на элемент списка. Два пункта:
- **"Изменить"** — открывает поля редактирования суммы и имени прямо в модалке/на месте. Сумма редактируется со знаком: положительное значение добавляет к остатку, отрицательное — вычитает. Сохранение пересчитывает остаток.
  - Для записей типа `total` ("Изменение баланса") поле имени скрывается/недоступно для редактирования — у таких записей имя всегда `null`.
  - Сохранение с суммой, равной `0`, запрещено (так же, как при создании новой сделки) — кнопка сохранения неактивна, пока сумма не заполнена ненулевым значением.
- **"Удалить"** — удаляет сделку сразу, без дополнительного подтверждения (кроме защищённой записи "Start", см. раздел "Структура и логика приложения").

## Функции и логика взаимодействия

При запуске приложения — экран счётчика.

Изначально блок счётчика показывает `0` в выбранной валюте (по умолчанию — `$`).

- Если **зажать число остатка** — открывается поле для ввода нового значения остатка вручную. После ввода и подтверждения создаётся новая запись `type = total` в базе (см. раздел "Структура и логика приложения") — **без дополнительного подтверждающего диалога**, ввод и есть подтверждение. Поле ввода имени операции при этом не используется (для `total`-записей `name` всегда `null`, см. раздел "Структура и логика приложения").
- Если **зажать символ валюты** — можно ввести свой символ/строку валюты (только текст, не числа). Это независимый способ смены валюты, отдельный от аналогичной настройки в экране настроек — оба места должны читать/писать одно и то же значение валюты, поэтому изменение в одном месте должно отражаться в другом.

При вводе значения через блок кнопок меняется значение нижней строки блока счётчика. Поле ввода имени операции работает независимо от поля суммы: стирание суммы (backspace до пустой строки) не сбрасывает уже введённое имя — оно остаётся в текстовом поле.

Ввод суммы ограничен 2 знаками после десятичной точки (как в денежных суммах) — ввод третьей цифры после точки игнорируется. Повторное нажатие "." после того, как точка уже есть в текущем вводе, также игнорируется.

Кнопка "Смена режима" переключает режим "Вычитание" — "Добавление". По умолчанию — режим "Вычитание". Уже введённое число при смене режима сохраняется — меняется только цвет и знак отображения, само значение не сбрасывается.
- Режим вычитания — введённая сумма вычитается из остатка.
- Режим добавления — введённая сумма добавляется к остатку.

Все изменения применяются только после нажатия кнопки "Введение". Если поле суммы пустое (или равно 0) — нажатие "Введение" ничего не делает, запись не создаётся, даже если поле имени операции уже заполнено (сохранить операцию только с именем, без суммы, нельзя). В этом состоянии кнопка "Введение" визуально отображается как неактивная (приглушённый/более светлый цвет фона по сравнению с активным состоянием, см. `enterBg` в прототипе-референсе).

После успешного нажатия "Введение" (когда запись создана) оба поля — сумма и имя операции — очищаются.

Переключение между экраном счётчика и экраном истории — нажатием на соответствующую часть кнопки-пилюли или свайпом.

Нажатие на кнопку добавления имени в списке истории открывает текстовое поле для ввода имени сделки.

Долгое нажатие на элемент списка истории открывает окно с пунктами "Изменить" и "Удалить" (логика — см. выше).

### Экран настроек
Открывается нажатием на иконку шестерёнки в шапке. Содержит:
- **Смена световой темы** (пресеты или кастомные цвета — на усмотрение реализации).
- **Округление баланса** — переключатель между отображением остатка с копейками/центами (по умолчанию) и округлением до целых.
- **Настройки уведомлений** — заглушка, без реальной логики на данном этапе.

Функционал экрана настроек в основном служит заготовкой на будущее — реализовать UI, но без сложной обвязки, если явно не сказано иное.

## Структура и логика приложения

`bill.db` — файл, содержащий историю всех сделок.

**Поля таблицы:**
- `id` — уникальный идентификатор. Устанавливается автоматически.
- `amount` — число с плавающей точкой. Сумма сделки, может быть положительной и отрицательной.
- `name` — строка. Название сделки. **Обеспечить защиту от SQL-инъекций** (использовать параметризованные запросы).
- `date` — целочисленное значение (тип `INTEGER` в SQLite). Дата и время сделки через `Date.now()` (Unix timestamp в миллисекундах). Устанавливается автоматически.
- `type` — строка, одно из двух значений:
  - `deal` — обычная сделка (доход/расход).
  - `total` — изменение общей суммы (ручная корректировка остатка).

Первая строка таблицы должна выглядеть так (`id` и `date` — автоматически):

| id | amount | name | date | type |
| --- | --- | --- | --- | --- |
| | 0 | Start | | total |

Эта первая строка ("Start") **защищена от удаления** через историю — она всегда должна оставаться точкой отсчёта для расчёта остатка. При долгом нажатии на неё в истории пункт "Удалить" должен быть недоступен (или отсутствовать).

Для последующих ручных корректировок остатка (когда пользователь зажимает число остатка и вводит новое значение) новая `total`-запись создаётся с `name = null`. В UI истории такая запись отображается с именем "Изменение баланса" — это значение выводится на основе `type = total`, а не хранится буквально в поле `name` (см. раздел "Экран История").

**Расчёт остатка** (верхняя строка блока счётчика): остаток = `amount` последней по времени записи с `type = total` + сумма `amount` всех записей `type = deal`, созданных **после** этой `total`-записи. Записи `deal`, созданные раньше последней `total`, в расчёт не входят (они относятся к предыдущему "периоду" баланса).

Запись вносится в `bill.db` при нажатии кнопки "Введение":
- Режим "Вычитание" — `amount` отрицательный, `type = deal`.
- Режим "Добавление" — `amount` положительный, `type = deal`.
- Если введённое значение пустое или равно `0` (в том числе если пользователь явно набрал "0") — нажатие "Введение" ничего не делает, запись не создаётся.

Ручной ввод остатка (зажатие числа на экране счётчика) создаёт новую строку с `type = total` и введённым значением `amount`. Записи `type = total` (кроме "Start") не защищены от удаления — их можно удалить через историю так же, как обычную сделку (см. модалку "Изменить/Удалить"). При удалении `total`-записи остаток пересчитывается заново по формуле выше (относительно предыдущей по хронологии `total`-записи и всех `deal` после неё).

**Часовой пояс:** дата и время сделок (`date`) хранятся и отображаются в локальном времени устройства пользователя.

**Индексация:** для истории (сортировка и фильтрация по дате) рекомендуется индекс по полю `date`.

**Хранение настроек приложения:** символ валюты, выбранная световая тема и настройка округления баланса — это глобальные настройки приложения, не привязанные к конкретной сделке. Их не нужно хранить в `bill.db`; использовать отдельное хранилище (например, AsyncStorage или отдельную таблицу `settings` в той же SQLite-базе — на усмотрение реализации).
